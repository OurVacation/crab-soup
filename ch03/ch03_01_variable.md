## 1. 🥘 핵심 재료 (Key Concepts)
Rust 에서는 변수를 선언하면 기본적으로 불변으로 선언된다.

### (1) 변수
Rust에서는 `let` 키워드로 변수를 선언하고 할당하는데, 여기서 중요한 건 한번 `let`으로 선언된 변수는 불변이라는거다. 가변변수로 만들고 싶으면 `let mut` 으로 선언해야한다.

이런식으로 한 이유는 코드의 가독성을 위함이며, `mut`으로 선언된 변수는 나중에 읽는 사람들에게 이 변수가 변동될 수 있다는 점을 알려줘서 이후에 이해하기 쉽도록 만들어준다.


### (2) 상수
- **정의:** : 항상 불변인 값
- **특징:**
	- `mut` 과 함께 사용 불가능, 항상 불변이다.
	- `const` 키워드로 선언되며, 값의 타입이 항상 명시되어야 한다.
	- 전역 스코프를 포함한 어느 스코프에서도 선언 가능하다. 따라서 많은 부분에서 알아야 하는 값에 접근 가능하다.
	- 반드시 상수 표현식으로만 설정되며, 런타임에서 계산되는 결과값은 안된다.
	- 선언된 스코프 내에서 프로그램 작동 전체 시간동안 유효하다.

### (3) 섀도잉
- **정의:** : 변수 이름 재사용
- **특징:**
	- 이미 선언된 변수와 동일한 이름으로 `let` 으로 선언
	- 이 과정에서 새 변수가 이전 변수를 가린다(shadowing)

## 2. 🍳 조리 과정 (Code & Analysis)
> **책의 예제 코드를 그대로 치지 말고, 변형하거나 주석을 달아 분석합니다.**

```rust
fn main() {
    // 1. x라는 이름의 변수 선언
    let x = 5;
    println!("처음 x: {}", x); // 출력: 5

    // 2. '섀도잉' 발생!
    // 이전에 있던 x는 '가려지고', x+1의 값을 가진 '새로운 x'가 선언됨.
    let x = x + 1;
    println!("섀도잉된 x: {}", x); // 출력: 6

    // 3. 스코프(Scope)와 섀도잉
    {
        // 이 블록 안에서 x를 또 섀도잉.
        let x = x * 2;
        println!("블록 안의 x: {}", x); // 출력: 12 (바깥 x인 6 * 2)
    } // 이 블록이 끝나면, 블록 안의 x는 사라진다.

    // 4. 블록 밖으로 나오면?
    // 블록이 끝나고, 다시 그전의 x (6)가 보이게 된다.
    println!("블록 밖의 x: {}", x); // 출력: 6
}
```

## 3. 🧂 깊은 맛 (Deep Dive)
💡 궁금증 : 섀도잉과 `mut` 의 차이
어차피 같은 이름으로 두 번씩 선언할거면 그냥 let mut x = 5; x = 6; 처럼 mut을 쓰면 안 되나??
: 사실 섀도잉을 하는 것과 `mut` 지정은 완전히 다르다.
1. **`mut` (Mutation / 변이):**
    - 변수 자체를 '가변'으로 만든다.
    - 값은 바꿀 수 있지만, **타입(Type)은 절대 바꿀 수 없다.**
    - 개념적으로 '같은 상자'의 내용물만 바꾸는 것.
2. **`let` (Shadowing / 섀도잉):**
    - `let`을 또 썼다는 건, **'완전히 새로운 변수'**를 만드는 것이다.
    - 이전 변수와 '이름만' 같다.
    - **전혀 다른 타입(Type)으로 선언할 수 있다.** (가장 중요한 차이!)
    - 개념적으로 '같은 이름표'를 새 상자에 붙이고, 이전 상자는 가려버리는 것이다.
